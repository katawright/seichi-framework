Personal mobile app solving household food waste ($50–100/month). Core problem:
perishables forgotten in the fridge, thrown out every 1–2 weeks. Every prior
system (whiteboard, Fridgely-style app) died at the item-adding step — friction
causes the list to drift, trust collapses, system abandoned.

**Why:** Recurring financial loss (~$600–1,200/year) plus guilt and unnecessary
takeout orders.

**Riskiest assumption:** Data entry can be made low-friction enough to outlast
prior attempts. This must be addressed as the first design spike.

**Stage:** Initiation — brief seeded 2026-06-10, working through remaining
sections (checklist + Gate 1 self-gate).

**How to apply:** The data-entry friction problem is the hill everything lives
or dies on. Any design discussion should start there. Partner participation is
a nice-to-have, not a requirement. [[user-profile]]
