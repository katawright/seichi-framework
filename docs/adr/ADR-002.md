# ADR-002: Replace Rework Severity Ladder with Guided Assessment Model

**Status:** Accepted

**Date:** 2026-03-26

---

## Decision

Replace the three-tier rework classification (Cosmetic / Significant /
Fundamental) with two independent diagnostic questions — _Does this change the
design?_ and _Does this affect the investment assumptions?_ — framed as guided
judgment rather than mechanical routing.

## Context

The Mid-Stage Discovery section used a rigid severity ladder to classify
discoveries that break something during active stage work:

| Severity        | ADR Required | Re-gate |
| --------------- | ------------ | ------- |
| **Cosmetic**    | No           | No      |
| **Significant** | Yes          | No      |
| **Fundamental** | Yes          | Yes     |

This model had four problems:

1. **Gap between tiers.** "Cosmetic" covered surface-level fixes (typos, button
   repositioning) while "Significant" jumped to design changes requiring an ADR.
   Non-trivial fixes with no design impact — a meaningful refactor that stays
   within the existing architecture and budget — had no natural home.

2. **False hierarchy.** The ladder treated ADR Required and Re-gate as a single
   escalation path, implying that design impact and investment impact always
   co-occur. They don't. A cost surprise can invalidate the investment thesis
   (warranting gate re-evaluation) without changing the design (no ADR needed).
   The No-ADR / Yes-Re-gate combination was valid but had no tier.

3. **Misleading terminology.** The decision tree routed "breaks something" to
   "Rework," implying the outcome is always additional work. But assessment
   might reveal that the right response is to stop the project or drop a
   requirement — less work, not more.

4. **Binary buckets for continuous spectrums.** Both design impact and
   investment impact are spectrums. What qualifies as "significant" depends on
   project context — remaining budget, schedule health, risk appetite,
   organizational culture. A rigid classifier can't accommodate this variation.

## Consequences

- The old tier names (Cosmetic, Significant, Fundamental) are retired as formal
  classifications. They may still appear as informal vocabulary in conversation.
- Practitioners answer two concrete diagnostic questions instead of debating
  tier boundaries. Both questions are framed as spectrums requiring judgment,
  not binary yes/no gates.
- Gate re-evaluation is decoupled from ADR requirements — all four combinations
  of the two questions are explicitly valid.
- The decision tree outcomes are renamed from "Rework / Scope amendment / Defer"
  to "Assess / Amend / Defer."
- Stage README callouts are updated to reference the new language.
- The Rework Cycles section in the Agentic Workflow Guide retains its title (the
  term is accurate for the verification-to-implementation defect-fix loop) but
  its content is updated to reference the two-question model.

## Alternatives considered

- **Add a fourth tier (e.g., "Minor") between Cosmetic and Significant:** Fills
  the gap but doesn't fix the false hierarchy or the continuous-spectrum
  problem. More tiers means more boundary debates.
- **Keep three tiers, widen "Cosmetic" to include larger no-design-impact
  fixes:** Partially addresses the gap but doesn't address the independent
  dimensions or the mechanical-routing problem.
